# Definition: Escrow (Trustless Work)

An Escrow represents a security deposit held using Trustless Work's decentralized smart contracts/accounts on Stellar, ensuring funds are released or claimed trustlessly.

## Key Attributes
- `id` (UUID): Unique internal identifier.
- `reservation_id` (UUID): References the reservation.
- `contract_address` (String): The Stellar smart contract address or escrow account generated by Trustless Work.
- `amount_usdt` (Numeric): The security deposit amount locked.
- `status` (Enum): `pending`, `funded`, `released`, `disputed`, `resolved`, `refunded`.
- `trustless_escrow_id` (String): The ID returned by Trustless Work's API.
- `created_at` (Timestamp): Creation date.
- `updated_at` (Timestamp): Modification date.

## State Transitions
- **pending**: Escrow configuration prepared but not yet funded.
- **funded**: Escrow contract verified funded on-chain.
- **released**: Funds released back to the Tenant after checkout (either approved by Host or auto-released after review window expiration).
- **disputed**: Escalation filed by Owner within the 72-hour review window.
- **resolved**: Dispute arbitrated, funds split or fully awarded.
- **refunded**: Booking cancelled before check-in, deposit returned.
